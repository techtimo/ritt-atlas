## What this project is

A single-page map/table viewer for German endurance riding events (Distanzritte), scraped hourly from the VDD MediaWiki. There is no build step, no npm, no bundler. The entire frontend is one HTML file; all dependencies are loaded from CDN.

## Files

- **`index.html`** — the page shell/markup; loads `includes/app.css`, `includes/app.js`, and the vendored MapLibre files from `includes/lib/`
- **`includes/`** — frontend assets: `app.css`, `app.js`, `favicon.svg`, and `lib/` (vendored MapLibre GL JS/CSS, no CDN)
- **`data.json`** — generated by the scraper; pretty-printed for readable git diffs; shape `{ scraped_at, events: [...] }`
- **`data.min.json`** — minified version of the same data; what the website fetches at runtime
- **`scraper/`** — Python tooling: `vdd_scrape.py` (fetches events from the VDD MediaWiki SMW API, geocodes missing coordinates via Nominatim, writes `data.json` + `data.min.json`), `notify_diff.py` (computes push-notification diffs), `send_test_push.py`, `test_notify_diff.py`, `requirements.txt`
- **`.github/workflows/update.yml`** — runs hourly; first does a cheap change-detection check (`--check-only`) against the MediaWiki RecentChanges API, then runs the full scrape only if changes are detected; commits both data files only when event data actually changed

## Running the scraper locally

```bash
pip install -r scraper/requirements.txt
python scraper/vdd_scrape.py
```

The scraper resolves `data.json`/`data.min.json` relative to the repo root (via `__file__`), so it can be run from any working directory. It makes two SMW `ask` queries (active events + Rittvorrat), then one batch `query&prop=info` call per 50 pages for `touched` timestamps, then Nominatim geocoding for any events missing wiki coordinates. It overwrites `data.json` in place.

## Viewing the app locally

```bash
python -m http.server
# open http://localhost:8000
```

A local HTTP server is required because `index.html` fetches `data.json` — browsers block `fetch()` on `file://` URLs.

## Architecture

**Data flow:** `vdd_scrape.py` → `data.json` (pretty, for diffs) + `data.min.json` (minified, served to browser) → `index.html` fetches `data.min.json` at startup, caches in `localStorage` for 3 hours.

**Frontend libraries (CDN, no local copies):**

- MapLibre GL 4 — map with German federal geodata vector tiles (`sgx.geodatenzentrum.de` basemap.de style); attribution collapsed by default (`attributionControl: { compact: true }`)
- Tabulator 6.3.1 — the sortable/filterable event table
- Google Fonts (Open Sans)

**Key JS globals in index.html:**

- `EVENTS` — flat array of all event objects (enriched from `VDD_DATA.events` with `id` and `event_types_arr`)
- `tbl` — the Tabulator instance (`#grid`)
- `markers` — `{ [id]: maplibregl.Marker }` map for all map markers
- `activeId` — currently highlighted event id
- `openPopup` — the currently open `maplibregl.Popup` instance (or `null`)

**Filter state** is held in module-level `let` variables (`filterText`, `userLat/userLon/radiusKm`, `selectedRegion`, `selectedEventTypes`, `selectedDistances`). All filtering goes through `applyFilters()` → `tbl.setFilter(eventVisible)`. Map markers are shown/hidden in the `dataFiltered` Tabulator event.

**Event focus** (`focusEvent`): highlights the table row, opens a MapLibre popup on desktop, or a fullscreen modal (`#popup-modal`) on mobile (≤860px breakpoint).

**Popup positioning (desktop only):** On mobile (≤860px) `.maplibregl-popup` is hidden via CSS and `#popup-modal` is used instead. On desktop, `focusEvent` uses two nested `requestAnimationFrame` calls: the first opens the popup (`showEventPopup`), the second measures the popup height and calls `map.flyTo()` with a dynamic `padding.top` calculated to keep the popup fully visible above the marker.

## data.json schema (per event)

Key fields: `id`, `wiki_title`, `vdd_url`, `name`, `subtitle`, `start_date`, `end_date`, `region`, `venue`, `lat`, `lon`, `organizer`, `event_types` (comma-separated string), `event_types_arr` (array, added at runtime), `efr/kdr/mdr/ldr/mtr/cei` (distance strings), `announcement_pdf`, `announcement_updated` (bool), `results_pdf`, `registration_pdf`, `status`, `first_edition_year`, `website`, `bemerkung` (free-text notes, ~44% of events), `ritt_bild` (photo filename, ~36% of events), `rittvorrat` (0/1), `wiki_touched` (YYYY-MM-DD of last wiki edit).

PDF and image URLs share the same base: `https://vdd-aktuell.de/mediawiki/index.php?title=Special:Redirect/file/<encoded_filename>`.

PDF links are constructed as `https://vdd-aktuell.de/mediawiki/index.php?title=Special:Redirect/file/<encoded_filename>`.

## CI

The GitHub Actions workflow runs every hour with two-stage change detection:

1. `python scraper/vdd_scrape.py --check-only` — queries `list=recentchanges` since `scraped_at`; exits early (no scrape, no commit) if the wiki has no new edits
2. If changes detected, runs the full scrape; `scraped_at` is only updated when the events array actually differs from the previous run, so `data.json` / `data.min.json` are committed only when real data changed
